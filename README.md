# Threat-Informed Vulnerability Prioritization: A Staged Framework Integrating KEV, EPSS, and MITRE ATT&CK

This repository contains the data and analysis scripts supporting the paper:

> Stephin Paul. *Threat-Informed Vulnerability Prioritization: A Staged Framework Integrating KEV, EPSS, and MITRE ATT&CK.* Submitted 2026.

---

## Repository Contents

| File | Description |
|---|---|
| `run_analysis.py` | Main analysis script. Downloads all required data and reproduces the results reported in Section 8 of the paper. |
| `analysis_results.xlsx` | Output file generated by the script. Contains a Summary sheet with all key metrics and a CVE Details sheet with per-CVE tier assignments. |

---

## How to Reproduce the Analysis

### Requirements

- Python 3.8 or later
- Internet connection (script downloads live data from public APIs)

### Install dependencies

```bash
pip install requests pandas openpyxl
```

### Run

```bash
python run_analysis.py
```

Runtime is approximately 2–5 minutes. The script will:

1. Download the CISA KEV catalog (JSON feed from cisa.gov)
2. Download current EPSS scores (CSV from epss.cyentia.com)
3. Download NVD bulk data for 2023 (via fkie-cad/nvd-json-data-feeds on GitHub, with fallback to the NVD REST API)
4. Apply Stage 1 (KEV check) and Stage 2 (EPSS threshold at 0.10) from the proposed framework
5. Compare results against a CVSS-only baseline (threshold ≥ 7.0)
6. Print results to the console and save `analysis_results.xlsx`

All data used in this analysis is publicly available at no cost.

|---|---|---|
| CVSS base scores | NIST NVD | https://nvd.nist.gov/ |

| EPSS scores | FIRST | https://www.first.org/epss/ |
| KEV catalog | CISA | https://www.cisa.gov/known-exploited-vulnerabilities-catalog |

## Important Note on Reproducibility

The script downloads **current** EPSS scores, not time-matched historical scores from 2023. EPSS scores for 2023 CVEs have had more than two years to absorb additional exploitation evidence, so current scores will differ from those that would have been available to a defender at the time of publication. This limitation is discussed in Section 8.4 of the paper. The results reported in the paper were generated using current-day scores as of early 2026.

To replicate the exact figures from the paper, run the script against the same data snapshot used during the study period (March 2026). The FIRST API supports date-specific EPSS queries for time-matched replication.

## Configuration

The following constants at the top of `run_analysis.py` can be adjusted:

```python
ANALYSIS_YEAR = 2023        # CVE publication year to analyse
EPSS_THRESHOLD = 0.10       # Stage 2 EPSS cutoff (0.10 = 10%)
CVSS_HIGH_THRESHOLD = 7.0   # CVSS-only baseline threshold
OUTPUT_FILE = "analysis_results.xlsx"
```
